1、Git基础——常用命令

  • git clone 地址 //将远程代码复制到本地
  • git pull //取回远程主机某个分支的更新，再与本地的指定分支合并。
  • git add (filename) //添加文件入本地库（一个个添加）
  • git rm filename //删除文件
  • git commit -m “提示信息” //添加本地文件入缓冲区（批量）
  • git push -u origin 分支名 //将缓冲区（本地分支）的内容提交到远程仓库（origin）（批量操作）
  • git status //展示工作区的状态
  • git remote -v //查看远程库的信息
  • git init // 把当前文件夹变成一个本地git库
  • touch filename // 新建文件
  • git remote add origin（自定义名字）地址 //将git项目地址取名为origin添加到远程库,r然后可用git push -u origin master命令

2、Git分支

创建、查看与切换
  • git branch -a //查看所有分支
  • git branch 分支名 //在master下创建分支
  • git checkout 分支名 //切换分支，切换到不同分支时，本地文件夹的内容也不同。
分支合并、冲突
  • 分支状态下合并到主干的步骤：
    1. git merge master
    2. git checkout master
    3. git merge 分支名
    4. git push -u origin master
  • 当两个分支都改动了同一个部分时，会发生冲突，后提交到主干的人需要手动确认内容，解决冲突
代码（版本）回溯
  1. git log -l 数字 // 查看前几步的操作
  2. 复制操作记录码
  3. git reset -hard 操作记录码 // 本地库强制退回到先前记录
  4. git push -f -u origin 分支名 // 强制提交，删除记录
    或者git revert 操作记录码 // 安全提交，保留记录
同时推送到多个远程库
  1. git remote add 库名 库地址
  2. git push -u 库名 分支名
