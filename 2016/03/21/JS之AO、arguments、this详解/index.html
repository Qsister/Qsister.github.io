    JS之AO、arguments、this详解

    图南
    2016-03-21
    前端

    作用域

    在JS函数嵌套中，变量的寻找方式： 从内到外，直到最外层（window）

    上述两例说明：
    1.JS代码至上而下执行
    2.但在整体运行前存在 词法分析过程，在此过程中，变量声明先被识别（察觉出变量已存在），但还未赋值

    ActiveObject（活动对象）

    词法分析过程：

    1. 分析参数
    2. 分析变量声明
    3. 分析函数声明

    具体步骤：

    1. 在函数运行前一瞬间生成AO
    2. 把函数声明的参数形成AO的属性
    3. 分析变量的声明！如var age，若它不在AO属性中，则把它添加到AO的属性，值是undefined
    4. 分析函数的声明！如function t(){}，则把函数赋给AO.t属性若之前t属性已存在，则被函数无情覆盖。

      例1

    分析过程：

    • 传入参数：AO.b=1
    • 变量声明：无
    • 函数声明：AO.a=function a();AO.b被覆盖为function b()
      执行过程：
    • alert(b);//function
    • b();//它的内部没有b，于是向外层寻找，找到了函数a()中的b,即function

      例2

    分析过程：

    • 传入参数：AO.b=1
    • 变量声明：无
    • 函数声明：AO.a=function a();
      执行过程：
    • alert(b);//1
    • b = function (){
      alert(b);
      }//赋值操作
    • b();//执行匿名函数，它的内部没有b，于是向外层寻找，找到了它自己，即弹出function(){alert(b)}

    总结：函数由内到外产生的AO链，叫做作用域链
